Where should user accounts & progress be stored?

This is a vanilla JS app with no backend server. Pick the storage approach that fits how you'll use the app.

A

Browser localStorage (Recommended)

User accounts and all exam history stored in the browser. Works offline, zero setup, instant. Data lives on this device/browser only — no cross-device sync.

  • ✅ No server needed
  • ✅ Works today, no extra files
  • ❌ Data lost if browser storage cleared
  • ❌ Can't log in from another device
B

JSON file on disk (via a tiny Node server)

A small Express/Node backend stores users.json and progress.json locally. Data persists across browsers/devices on the same machine.

  • ✅ Survives browser clears
  • ✅ Could be shared on a local network
  • ❌ Requires running a Node server
  • ❌ More setup and files to maintain
C

Firebase / Supabase (cloud backend)

Free-tier cloud database. Real cross-device login, proper auth, data never lost. Requires internet and a free account setup.

  • ✅ Real multi-device login
  • ✅ Data always safe
  • ❌ Needs internet connection
  • ❌ Account setup + API keys required
